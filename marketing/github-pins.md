# GitHub profile + pinned repos guidance

This repo is a static site, so GitHub profile README edits and pinned-repo selection must be done in GitHub UI. This file gives you a consistent “what to pin + what to write” checklist.

## 1) Keep your GitHub profile story consistent

Copy/paste this into your GitHub profile README (or wherever GitHub shows your main intro):

**Engineering professional + Industrial Engineering student at the intersection of industry operations, automation/AI—documented through case studies and writing.**

Live site: https://jassimgafoor.github.io/

## 2) What to pin (pick by narrative fit)

Pin 2–3 repositories that best prove your positioning:

1. Automation in operations (scripts, data pipelines, integrations, workflow tooling).
2. AI-powered workflow / productivity (assistants, evaluation harnesses, or automation around AI outputs).
3. Practical automation in operations (integrations, data pipelines, workflow tooling; emphasize problem + outcome).

If you have only 2 strong repos, pin 2. If you have more, rotate quarterly based on what you’re posting on LinkedIn that month.

## 3) Pinned repo description template

Use this description format in each pinned repo:

Built for: [industry/workflow]

Problem: [what was slow/error-prone]

What I made: [tooling/approach]

Impact: [result or measurable outcome]

Where it’s documented: https://jassimgafoor.github.io/case-study.html (or https://jassimgafoor.github.io/blog.html)

## 4) Link hygiene

- In each pinned repo’s README, add a short “See also” line pointing to the relevant section of your site.
- Keep repo names/descriptions focused on outcomes, not just technologies.
